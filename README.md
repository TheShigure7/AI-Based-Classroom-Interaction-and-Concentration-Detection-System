# AI课堂互动与摸鱼检测系统

本目录为项目初始骨架，按“后端识别服务、前端展示、模型资源、数据存储、脚本工具、文档材料”进行划分，便于后续逐步补充代码与实验内容。

## 目录说明

- `backend/`：后端识别与接口服务
- `frontend/`：前端页面与可视化界面
- `models/`：模型权重、训练配置与导出模型
- `data/`：样本数据、标注文件、数据库与运行日志
- `scripts/`：环境初始化、训练、启动、测试脚本
- `docs/`：方案文档、图表素材、答辩材料
- `tests/`：测试代码与测试样例

## 当前阶段建议

建议先实现以下最小可用版本：

1. `backend/app/main.py` 提供后端入口
2. `backend/app/services/camera/` 完成摄像头取流
3. `backend/app/services/detection/` 完成 YOLO 目标检测
4. `backend/app/services/analysis/` 完成低头和举手规则判断
5. `frontend/` 完成实时监控页面

## 最小运行说明

### 1. 一键安装后端依赖

```powershell
.\一键安装后端依赖.bat
```

或手动执行：

```powershell
.\.venv\Scripts\python.exe -m pip install -r .\backend\requirements.txt
```

### 2. 一键打开摄像头检测

```powershell
.\一键启动摄像头检测.bat
```

说明：

- 当前默认输入源为本机 `0` 号摄像头
 - 若切换到网络视频流，可设置环境变量 `CLASSROOM_VIDEO_SOURCE=http://你的IP:8080`
 - 程序会自动把这类根地址解析为 `http://你的IP:8080/video`
- 默认加载 `YOLO` 课堂检测模型
- 当前可检测 `student(person)` 和 `phone(cell phone)`
- 当前已接入第一版举手识别，会在检测框附近显示 `hand raised`
- 当前已接入第一版低头识别，会在检测框附近显示 `head down`
- 当前已接入第一版疑似看手机识别，会在检测框附近显示 `phone risk`
- 当前已接入匿名学生跟踪，会在框上方显示 `1 | 86` 这类编号和专注度
- 学生框颜色规则：正常绿色、举手蓝色、低头黄色、疑似看手机红色
- 当前已接入专注度评分与班级平均专注度统计
- 首次运行时，`Ultralytics` 可能会自动下载默认模型权重
- 首次运行举手识别时，`MediaPipe` 会自动下载姿态模型文件
- 已兼容当前项目中文目录路径，不需要手动移动模型文件
- 按 `q` 退出预览窗口

补充：

- `http://10.62.111.134:8080` 是网页首页，不是直接视频流
- 实际可被 OpenCV 打开的流地址是 `http://10.62.111.134:8080/video`

### 额外推荐：单文件自动运行

```powershell
.\一键运行课堂检测.bat
```

这个入口会：

1. 检查项目虚拟环境是否存在
2. 若不存在则先自动安装后端依赖
3. 自动启动摄像头检测

### 3. 一键启动后端服务

```powershell
.\一键启动后端服务.bat
```

启动后可访问：

- 健康检查：`http://127.0.0.1:8000/api/v1/health`
- 接口文档：`http://127.0.0.1:8000/docs`
