# 实时检测页接口字段设计

## 1. 文档目标

本文件用于冻结“实时检测页”第一阶段开发所需的前后端接口、字段命名和组件数据结构。

适用范围：
- 前端实时检测页
- 后端会话管理与实时状态输出
- 实时视频流
- WebSocket 推送
- 异常截图列表

本阶段目标：
- 前端可完成实时检测页第一版
- 后端可完成最小可用检测服务化
- 前后端能完成第一轮联调

---

## 2. 页面范围

实时检测页包含以下功能区：

1. 顶部控制区
- 当前视频源
- 切换视频源
- 开始检测
- 停止检测
- 当前状态
- 检测时长

2. 左侧实时视频区
- 视频流显示
- 检测框
- 学生状态标签

3. 右侧实时统计区
- 学生人数
- 平均专注度
- 低专注人数
- 各类行为人数和占比
- 推理 FPS

4. 底部异常截图区
- 异常截图列表
- 时间
- 学生编号
- 异常类型
- 当前分数

---

## 3. 第一阶段必须实现的接口

第一阶段只做这 6 个接口：

1. `POST /api/v1/session/start`
2. `POST /api/v1/session/stop`
3. `GET /api/v1/session/status`
4. `GET /api/v1/classroom/current`
5. `GET /api/v1/video/stream`
6. `WS /ws/classroom/realtime`

说明：
- 不在第一阶段实现检测记录页接口
- 不在第一阶段实现分析页接口
- 不在第一阶段实现 AI 接口

---

## 4. 接口设计

## 4.1 开始检测

### 接口
`POST /api/v1/session/start`

### 请求体

```json
{
  "video_source": "0",
  "enable_pose_analysis": true,
  "save_alert_snapshots": true
}
```

### 字段说明

- `video_source`
  - 类型：`string`
  - 含义：视频源
  - 示例：
    - `"0"`：本地摄像头
    - `"http://10.62.111.134:8080"`：网络视频流根地址

- `enable_pose_analysis`
  - 类型：`boolean`
  - 含义：是否开启姿态分析

- `save_alert_snapshots`
  - 类型：`boolean`
  - 含义：是否保存异常截图

### 返回体

```json
{
  "success": true,
  "message": "session started",
  "session": {
    "session_id": "SESSION_20250706_001",
    "status": "running",
    "video_source": "0",
    "started_at": "2025-07-06T10:15:23"
  }
}
```

### 状态值约定

- `idle`
- `running`
- `paused`
- `stopped`
- `error`

---

## 4.2 停止检测

### 接口
`POST /api/v1/session/stop`

### 请求体

可为空：

```json
{}
```

### 返回体

```json
{
  "success": true,
  "message": "session stopped",
  "session": {
    "session_id": "SESSION_20250706_001",
    "status": "stopped",
    "stopped_at": "2025-07-06T10:45:18",
    "duration_seconds": 1795
  }
}
```

---

## 4.3 获取当前检测会话状态

### 接口
`GET /api/v1/session/status`

### 返回体

```json
{
  "session_id": "SESSION_20250706_001",
  "status": "running",
  "video_source": "0",
  "started_at": "2025-07-06T10:15:23",
  "duration_seconds": 532,
  "enable_pose_analysis": true,
  "save_alert_snapshots": true
}
```

### 字段说明

- `session_id`
  - 当前检测会话编号

- `status`
  - 当前检测状态

- `video_source`
  - 当前视频源

- `duration_seconds`
  - 当前检测已运行时长，单位秒

## 4.5 视频流接口

### 接口
`GET /api/v1/video/stream`

### 说明

返回类型：
- `multipart/x-mixed-replace`
- `MJPEG`

前端使用方式：

```html
<img src="http://127.0.0.1:8000/api/v1/video/stream" />
```

### 注意事项

- 第一阶段直接输出已绘制框和标签的视频流
- 前端不在第一阶段自行画框
- 这样能最快完成页面演示

后续如果要做更复杂交互，可改成：
- 视频裸流 + 前端单独画框层

但不是当前阶段重点。

## 4.6 WebSocket 实时推送

### 接口
`WS /ws/classroom/realtime`

### 推送频率建议
- 每秒 `1~3` 次即可
- 不建议每帧都推

### 推送数据结构

推送结构与 `GET /api/v1/classroom/current` 保持一致，建议最小化字段后推送：

  "session_id": "SESSION_20250706_001",

  "performance": {
    "inference_fps": 5.2,
    "display_fps": 24.0

  "summary": {
    "student_count": 12,
    "average_attention": 76,
    "low_attention_count": 3,
    "behavior_counts": {
      "hand_raised": 2,
      "head_down": 2,
      "phone_risk": 1,
      "sleeping": 1,
      "talking_risk": 2
    },
    "behavior_rates": {
      "hand_raised": 0.17,
      "head_down": 0.17,
      "phone_risk": 0.08,
      "sleeping": 0.08,

## 5. 字段命名规范

### 5.1 命名规则

统一使用：
- `snake_case`

原因：
- 当前后端 Python 为主
- 现有数据结构更适合 `snake_case`

前端如需转为 camelCase，可在前端统一适配，但后端返回统一用 `snake_case`。

### 5.2 布尔字段统一

推荐：
- `running`
- `is_low_attention`

不混用：
- `lowAttention`
- `low_attention_flag`
- `low`

### 5.3 枚举值统一

`event_type` 统一使用：
- `hand_raised`
- `head_down`
- `phone_risk`
- `sleeping`
- `talking_risk`
- `low_attention`

## 7. 第一阶段联调顺序

推荐联调顺序如下：

### 第一步
前端先用假数据完成实时检测页静态布局

### 第二步
后端实现：
- `POST /session/start`
- `POST /session/stop`
- `GET /session/status`
- `GET /classroom/current`

### 第三步
前端接通：
- 顶部控制区
- 右侧统计区

### 第四步
后端实现：
- `GET /video/stream`

### 第五步
前端接通视频区

### 第六步
后端实现：
- `WS /ws/classroom/realtime`

### 第七步
前端用 WebSocket 刷新：
- 统计数据
- 异常截图列表

完成这七步，就算第一阶段闭环。

---

## 8. 当前阶段不做的内容

第一阶段不做：
- 检测记录详情页接口
- 分析页面接口
- 设置页完整接口
- AI 对话接口
- 用户登录
- 权限管理
- API 网关

---

## 9. 结论

实时检测页第一阶段开发应围绕以下能力展开：

1. 启动/停止检测
2. 查看当前状态
3. 显示实时视频
4. 显示实时统计
5. 显示异常截图
6. 实时 WebSocket 推送

当前阶段最重要的是：
- 冻结这份字段设计
- 前后端按同一字段同步开发
- 先打通“实时检测页”最小可演示闭环
