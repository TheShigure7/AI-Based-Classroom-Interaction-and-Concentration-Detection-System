# Image2.0 页面预览图生成指导

## 1. 使用目标

本文件用于指导使用生图类 AI 工具生成本项目的界面预览图，供后续前端设计和开发参考。

项目名称：
- AI课堂互动与摸鱼检测系统

建议先生成以下 4 个页面预览图：
- 实时检测页
- 检测记录页
- 分析页面
- 设置页

如果时间有限，优先生成：
- 实时检测页

---

## 2. 生成原则

生成界面图时，不要把它当作普通海报，而要当作真实的“监测系统后台界面”。

要求：
- 风格偏真实 Web 管理系统
- 信息密度中高
- 布局清晰，不能过于花哨
- 颜色要区分状态
- 页面中有图表、卡片、表格、视频区、按钮等典型 UI 元素
- 适合课堂监测、数据分析、异常预警场景

建议风格关键词：
- modern web dashboard
- classroom monitoring system
- admin panel
- data visualization
- clean enterprise UI
- real-time analytics

不建议风格：
- 赛博朋克
- 过度科幻
- 海报风
- 手机 App 风
- 插画风
- 纯线框图风

---

## 3. 统一视觉规范

所有页面建议保持统一视觉语言。

### 3.1 色彩建议

主色：
- 深蓝灰
- 冷白
- 浅灰蓝

状态色建议：
- 正常：绿色
- 举手：蓝色
- 低头：黄色
- 手机风险：红色
- 睡觉风险：紫红色
- 疑似交谈：橙色

### 3.2 版式建议

- 顶部有导航栏
- 左侧可有菜单栏，也可用顶部 tab
- 主体内容以卡片、图表、视频区为主
- 每个页面要像真实软件截图，不像设计稿拼贴

### 3.3 字体和信息

生图模型对中文小字经常不稳定，因此建议：
- 页面标题可写中文
- 大量小字说明尽量用短词
- 表格文字可简化
- 如果模型生成文字容易乱码，可以接受，用结构为主

推荐标题文本：
- AI课堂互动与摸鱼检测系统
- 实时检测
- 检测记录
- 分析页面
- 设置
- AI助手

```text
整体配色为深蓝灰、冷白、浅灰蓝，辅以绿色、蓝色、黄色、红色、橙色、紫红色作为行为状态高亮色。
界面强调“课堂实时监测、行为识别、专注度分析、异常预警”的产品定位。
请生成接近真实 SaaS 平台的 UI 截图效果，而不是海报、线框图或插画。
```

## 5. 通用负面提示词

每次生成页面时，建议都加上负面提示词，减少偏题。

```text
不要手机界面，不要海报，不要插画，不要卡通，不要科幻 HUD，不要游戏 UI，不要赛博朋克，不要过度发光，不要极端未来感，不要拟物风，不要杂乱无章，不要低信息量，不要纯线框草图，不要大面积留白，不要把页面做成宣传页。
```

## 6. 页面一：实时检测页

这是最重要的一页，建议先生成。

### 6.1 页面功能目标

需要体现：
- 中央是实时检测视频区域
- 视频中有学生检测框
- 框旁有专注度和行为标签
- 右侧有实时统计卡片
- 底部有异常截图列表
- 顶部有开始检测、停止检测等按钮

### 6.2 页面结构要求

页面应包含：
- 顶部导航栏
- 页面标题“实时检测”
- 中间左侧大视频区域
- 中间右侧统计信息面板
- 底部异常截图列表
- 顶部或右上角控制按钮

视频区域里建议有：
- 学生检测框
- 学生编号
- 专注度数值
- hand raised / head down / phone risk / sleep risk / talking risk 等标签

右侧统计区建议有：
- 检测时长
- 当前学生人数
- 平均专注度
- 低专注人数
- 举手人数
- 低头人数
- 手机风险人数
- 睡觉风险人数
- 疑似交谈人数
- 当前推理 FPS

底部异常截图区建议有：
- 多张异常缩略图
- 时间
- 异常类型
- 学生编号

### 6.3 推荐提示词

```text
设计一张高保真桌面端 Web 后台界面截图，页面名称为“实时检测”，属于“AI课堂互动与摸鱼检测系统”。

这是一个课堂行为监测系统的主界面。
页面顶部有系统标题、导航栏、当前视频源、检测时长、开始检测按钮、停止检测按钮。

页面主体左侧是大尺寸实时视频监测区域，显示教室画面，多个学生被检测框标注出来。
每个学生框旁边显示编号、专注度分数和状态标签，例如 hand raised、head down、phone risk、sleep risk、talking risk。
框颜色区分不同状态：正常绿色、举手蓝色、低头黄色、手机红色、睡觉紫红色、疑似交谈橙色。

页面右侧是实时统计面板，使用卡片式布局，显示当前学生人数、平均专注度、低专注人数、举手人数、低头人数、手机风险人数、睡觉风险人数、疑似交谈人数、推理 FPS。

页面底部有异常截图列表区域，显示多张异常缩略图卡片，每张卡片带时间、异常类型、学生编号。

整体风格是现代企业级数据监测平台，简洁、真实、专业、可开发，深蓝灰与冷白配色，局部用彩色状态标签点缀。
请生成像真实 SaaS 平台产品截图一样的界面，不要做成海报。
```

### 6.4 二次优化提示词

如果第一次生成后视频区不够突出，可以追加：

```text
让中间的视频监测区域更大、更突出，占据页面主要视觉中心；右侧统计区保持紧凑专业。
```

如果第一次生成后太像通用数据后台，可以追加：

```text
加强“课堂监测”特征，在视频区域明确体现教室场景、学生检测框、行为识别标签和异常状态。
```

## 7. 页面二：检测记录页

### 7.1 页面功能目标

需要体现：
- 异常记录列表
- 时间筛选
- 异常类型筛选
- 截图预览
- 详情面板

### 7.2 页面结构要求

页面包含：
- 顶部导航栏
- 标题“检测记录”
- 上方筛选栏
- 中间记录表格或卡片列表
- 右侧详情预览或弹窗式详情

筛选栏建议有：
- 日期范围
- 异常类型
- 学生编号
- 搜索框

记录区建议展示：
- 时间
- 异常类型
- 学生编号
- 专注度
- 截图缩略图
- 状态标签

### 7.3 推荐提示词

```text
设计一张高保真桌面端 Web 后台界面截图，页面名称为“检测记录”，属于“AI课堂互动与摸鱼检测系统”。

这是课堂检测系统中的历史记录页面。
顶部有系统导航栏和页面标题。
页面上方有筛选区域，包含日期范围选择、异常类型筛选、学生编号筛选、搜索框。

页面中间是检测记录列表，采用表格和卡片结合的企业后台风格。
每条记录包含时间、异常类型、学生编号、专注度分数、异常截图缩略图、状态标签。

页面右侧有详情预览区，展示选中记录的大图预览、详细信息和异常说明。

整体风格保持与实时检测页一致，现代、专业、真实、简洁，偏企业级管理平台，适合数据记录与异常追踪场景。
```

## 8. 页面三：分析页面

### 8.1 页面功能目标

需要体现：
- 各类统计图表
- 专注度趋势
- 活跃度分析
- 各行为占比

### 8.2 页面结构要求

页面包含：
- 顶部导航栏
- 标题“分析页面”
- 多个图表卡片

建议图表：
- 平均专注度折线图
- 低专注人数趋势图
- 各类行为次数柱状图
- 行为占比饼图
- 活跃度趋势图

建议补充信息卡片：
- 今日平均专注度
- 风险峰值时间段
- 最多出现的异常行为

### 8.3 推荐提示词

```text
设计一张高保真桌面端 Web 后台界面截图，页面名称为“分析页面”，属于“AI课堂互动与摸鱼检测系统”。

这是课堂行为分析与专注度统计页面。
页面顶部有导航栏和标题。
主体区域由多个数据图表卡片组成，包含平均专注度折线图、低专注人数趋势图、行为次数柱状图、行为占比饼图、课堂活跃度趋势图。

页面上方或左侧有统计卡片，显示今日平均专注度、异常行为总数、风险峰值时间段、最常见异常行为。

整体视觉风格应真实、专业、现代，像成熟的数据分析平台，图表布局清晰，信息密度较高，但层次分明。
保持与实时检测页统一的深蓝灰、冷白、浅灰蓝配色体系。
```

## 9. 页面四：设置页

### 9.1 页面功能目标

需要体现：
- 视频源设置
- 历史视频源记录
- 检测参数开关
- 提醒和邮件配置

### 9.2 页面结构要求

页面包含：
- 顶部导航栏
- 标题“设置”
- 多个设置分组卡片

设置分组建议：
- 视频源设置
- 检测参数设置
- 提醒设置
- 系统状态

视频源设置建议展示：
- 本地摄像头选项
- 手机/IP 视频源输入框
- 历史视频源记录列表
- 切换按钮

检测参数建议展示：
- 姿态分析开关
- 睡觉识别开关
- 疑似交谈识别开关
- 低专注阈值滑块
- 保存异常截图开关

### 9.3 推荐提示词

```text
设计一张高保真桌面端 Web 后台界面截图，页面名称为“设置”，属于“AI课堂互动与摸鱼检测系统”。

这是系统设置页面。
顶部有导航栏和标题。
页面主体是多个卡片式设置分区，包含视频源设置、检测参数设置、提醒设置、系统状态。

视频源设置区包含本地摄像头选项、网络视频源输入框、历史视频源记录列表、切换按钮。
检测参数区包含多个开关、下拉框、滑块，例如姿态分析开关、睡觉识别开关、疑似交谈识别开关、低专注阈值、是否保存异常截图。
提醒设置区包含实时提醒开关、邮件总结开关、邮箱配置输入框。
系统状态区显示当前后端状态、模型状态、日志摘要。

整体风格要像真实企业后台设置页，专业、整洁、可开发，不要做成普通表单页截图，要有层次感和系统感。
```

- 聊天面板

### 10.2 结构建议

### 10.3 推荐提示词

```text
主界面仍然是课堂监测系统后台，右侧展开一个 AI 助手面板。
面板内包含聊天记录区、输入框、发送按钮、实时提醒卡片、每日总结入口。
聊天内容围绕课堂状态分析、风险提醒、专注度总结展开。

整体风格应与主系统保持一致，现代、专业、真实，像已经集成在监测系统中的 AI 助理功能，而不是单独的聊天软件界面。
```

## 11. 推荐生成顺序

建议按下面顺序生成：

1. 实时检测页
2. 分析页面
3. 检测记录页
4. 设置页
5. AI助手抽屉

原因：
- 实时检测页最核心，后续做界面主要参考它
- 分析页面最能体现系统价值
- 检测记录和设置页相对容易补

## 12. 如果生成效果不理想，如何继续提示

### 情况 1：太像通用后台，没有课堂特色

追加：

```text
增强课堂监测场景特征，明确展示教室视频画面、学生检测框、行为识别标签、专注度分数和异常截图列表。
```

### 情况 2：太像海报，不像系统

追加：

```text
请严格生成真实 Web 管理系统界面截图风格，强调卡片、图表、表格、按钮、控制栏、状态栏等软件界面元素，不要做成宣传海报。
```

### 情况 3：颜色过花

追加：

```text
收敛色彩，使用深蓝灰、白色、浅灰蓝作为基础，只保留少量高亮状态色，不要过度炫彩。
```

### 情况 4：排版太空

追加：

```text
提高信息密度，让页面更像真实监测系统后台，增加统计卡片、图表、列表和数据块，减少无意义留白。
```

### 情况 5：看起来太假、太概念图

追加：

```text
请生成更接近真实 SaaS 平台产品截图的界面效果，增强可开发性和真实软件结构感。
```

### 第二步
再加上某个页面的专属提示词

### 第三步
最后再加“通用负面提示词”

组合后就是一次完整生成请求。

## 14. 建议输出参数

如果工具支持，建议：
- 比例：16:9
- 分辨率：`1536x1024` 或 `1792x1024`
- 风格强度：中等
- 细节程度：高

- 每次生成 2 到 4 张
- 从中挑最接近系统感的一张
